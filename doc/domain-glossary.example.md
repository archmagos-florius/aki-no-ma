# Example: FocusTasks (todo app with focus feature)
```markdown
## AppName  
- FocusTasks

## User  
- A person who owns tasks and projects in the app.

## Project  
- A logical grouping of tasks (e.g., "School", "Side Project"). Each project belongs to a single user.

## Task  
- A unit of work the user wants to complete. Has title, description, due date, status, and optional reminder.

## Task Status  
- The state of a task. Allowed values: `pending`, `in_progress`, `done`, `archived`.

## Focus Session  
- A timed period (e.g., 25 minutes) where the user works on a selected set of tasks without distraction.

## Reminder  
- A scheduled notification attached to a task (e.g., "today at 18:00").

## Inbox  
- Default project where new tasks go if the user doesnâ€™t pick a project.

## Tag  
- A label the user can attach to tasks (e.g., "urgent", "school").

## Overdue Task  
- A task whose due date is in the past and status is not `done` or `archived`.
```
